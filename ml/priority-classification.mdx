## Overview

The priority classification system automatically assigns priority levels to incidents based on the content of their descriptions. The system uses a keyword matching algorithm to categorize incidents into three priority levels: **alta** (high), **media** (medium), and **baja** (low).

## Priority Levels

The system recognizes three priority levels based on the severity and urgency of the incident:

| Priority | Description | Use Case |
|----------|-------------|----------|
| **alta** | High priority - requires immediate attention | Emergency situations, security threats, accidents |
| **media** | Medium priority - needs attention soon | Environmental issues, infrastructure problems |
| **baja** | Low priority - can be addressed later | Minor disturbances, non-urgent complaints |

## Keyword Matching Algorithm

The classification system uses a simple yet effective keyword-based approach. Each incident description is analyzed for the presence of specific keywords that indicate the severity level.

### Priority Keywords

The system maintains a predefined dictionary of keywords for each priority level:

```python
palabras_prioridad = {
    "alta": ["humo", "incendio", "arma", "robo", "accidente", "caer"],
    "media": ["ruido", "contaminación", "basura", "luz"],
    "baja": ["vecinos", "madrugada", "música"]
}
```

#### High Priority Keywords (alta)
- **humo** - Smoke (potential fire hazard)
- **incendio** - Fire
- **arma** - Weapon
- **robo** - Robbery/theft
- **accidente** - Accident
- **caer** - Fall (person or object)

#### Medium Priority Keywords (media)
- **ruido** - Noise
- **contaminación** - Pollution
- **basura** - Trash/garbage
- **luz** - Light (lighting issues)

#### Low Priority Keywords (baja)
- **vecinos** - Neighbors
- **madrugada** - Early morning/late night
- **música** - Music

## Classification Process

The system follows a hierarchical matching approach:

1. **Text Normalization**: Convert the incident description to lowercase
2. **High Priority Check**: Search for high priority keywords first
3. **Medium Priority Check**: If no high priority keywords found, search for medium priority keywords
4. **Default Assignment**: If no keywords match, assign low priority by default

```python
def asignar_prioridad(texto):
    texto = texto.lower()
    for palabra in palabras_prioridad["alta"]:
        if palabra in texto:
            return "alta"
    for palabra in palabras_prioridad["media"]:
        if palabra in texto:
            return "media"
    return "baja"
```

### Algorithm Characteristics

- **Hierarchical**: Checks high priority first, then medium, then defaults to low
- **First Match Wins**: Returns immediately upon finding the first matching keyword
- **Case Insensitive**: All text is converted to lowercase before matching
- **Substring Matching**: Keywords can be found anywhere in the description
