## Your First API Call

This guide will walk you through the essential workflow: registering a user, logging in to get an access token, and creating your first incident report.

<Note>
  This quickstart assumes you have the API running locally on `http://localhost:8000`. If you haven't installed it yet, check out the [Installation Guide](/installation).
</Note>

## Prerequisites

Before you begin, ensure you have:

- The API server running (`uvicorn main:app --reload`)
- A tool to make HTTP requests (curl, Postman, or HTTPie)
- Your database properly configured and migrated

## Step-by-Step Workflow

<Steps>
  <Step title="Register a New User">
    First, create a new user account. The registration endpoint is public and doesn't require authentication.

    ```bash Register User
    curl -X POST "http://localhost:8000/register" \
      -H "Content-Type: application/json" \
      -d '{
        "nombre": "Juan",
        "apellidos": "Pérez García",
        "correo_electronico": "juan.perez@example.com",
        "telefono": "+52 55 1234 5678",
        "contrasena": "SecurePassword123!"
      }'
    ```

    **Response:**

    ```json
    {
      "usuario": {
        "id": 1,
        "nombre": "Juan",
        "correo_electronico": "juan.perez@example.com",
        "rol": "ciudadano",
        "fecha_creacion": "2026-03-03T10:30:00"
      },
      "message": "Usuario registrado correctamente"
    }
    ```

    <Note>
      The password is automatically hashed using bcrypt before storage. The default role is `ciudadano` (citizen).
    </Note>
  </Step>

  <Step title="Login and Get Access Token">
    Now authenticate with your credentials to receive a JWT access token.

    ```bash Login
    curl -X POST "http://localhost:8000/login" \
      -H "Content-Type: application/json" \
      -d '{
        "correo_electronico": "juan.perez@example.com",
        "contrasena": "SecurePassword123!"
      }'
    ```

    **Response:**

    ```json
    {
      "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "token_type": "bearer"
    }
    ```

    <Warning>
      Save this token! You'll need it for authenticated endpoints. The token includes your user ID, name, and role.
    </Warning>
  </Step>

  <Step title="Create Your First Incident">
    Report an urban incident with geolocation data. This endpoint doesn't require authentication in the current implementation.

    ```bash Create Incident
    curl -X POST "http://localhost:8000/incidente/" \
      -H "Content-Type: application/json" \
      -d '{
        "titulo": "Semáforo descompuesto en crucero principal",
        "descripcion": "El semáforo no funciona desde hace dos días, causando congestionamiento y riesgo de accidentes",
        "imagen": "https://example.com/images/semaforo-roto.jpg",
        "latitud": 19.432608,
        "longitud": -99.133209,
        "estado": "pendiente",
        "usuario_id": 1,
        "categoria_id": 1
      }'
    ```

    **Response:**

    ```json
    {
      "id": 1,
      "titulo": "Semáforo descompuesto en crucero principal",
      "descripcion": "El semáforo no funciona desde hace dos días, causando congestionamiento y riesgo de accidentes",
      "imagen": "https://example.com/images/semaforo-roto.jpg",
      "latitud": 19.432608,
      "longitud": -99.133209,
      "estado": "pendiente",
      "usuario_id": 1,
      "categoria_id": 1,
      "fecha_reporte": "2026-03-03T10:35:00"
    }
    ```

    <Note>
      The `categoria_id` should reference an existing category. Common categories include infrastructure (1), security (2), health (3), and environment (4).
    </Note>
  </Step>

  <Step title="List All Incidents">
    Retrieve a list of reported incidents with pagination support.

    ```bash List Incidents
    curl -X GET "http://localhost:8000/incidentes/?skip=0&limit=10"
    ```

    **Response:**

    ```json
    [
      {
        "id": 1,
        "titulo": "Semáforo descompuesto en crucero principal",
        "descripcion": "El semáforo no funciona desde hace dos días...",
        "latitud": 19.432608,
        "longitud": -99.133209,
        "estado": "pendiente",
        "fecha_reporte": "2026-03-03T10:35:00",
        "usuario_id": 1,
        "categoria_id": 1
      }
    ]
    ```
  </Step>

  <Step title="Get Incident by ID">
    Retrieve details of a specific incident.

    ```bash Get Single Incident
    curl -X GET "http://localhost:8000/incidente/1"
    ```

    Returns the same structure as the create response.
  </Step>
</Steps>

## ML-Powered Features

### Get Priority Classification

The API uses machine learning to automatically classify incident priorities based on description keywords.

```bash Get Prioritized Incidents
curl -X GET "http://localhost:8000/incidentes_prioridad"
```

**Response:**

```json
[
  {
    "id": 1,
    "descripcion": "Incendio en edificio abandonado",
    "fecha_reporte": "2026-03-03T10:00:00",
    "prioridad": "alta"
  },
  {
    "id": 2,
    "descripcion": "Basura acumulada en la esquina",
    "fecha_reporte": "2026-03-03T11:00:00",
    "prioridad": "media"
  }
]
```

**Priority Classification Rules:**

- **Alta (High)**: Keywords like "incendio", "arma", "robo", "accidente", "caer", "humo"
- **Media (Medium)**: Keywords like "ruido", "contaminación", "basura", "luz"
- **Baja (Low)**: Keywords like "vecinos", "madrugada", "música"

### Detect Critical Zones

Identify geographical areas with high incident concentration using DBSCAN clustering.

```bash Detect Critical Zones
curl -X GET "http://localhost:8000/ml/zonas-criticas"
```

**Response:**

```json
{
  "total_incidentes": 45,
  "eps_usado": 0.00234,
  "total_zonas": 3,
  "zonas_criticas": [
    {
      "zona_id": 0,
      "cantidad_incidentes": 15,
      "lat_centro": 19.4326,
      "lng_centro": -99.1332
    },
    {
      "zona_id": 1,
      "cantidad_incidentes": 12,
      "lat_centro": 19.4285,
      "lng_centro": -99.1278
    }
  ]
}
```

<Note>
  The `eps_usado` value is automatically calculated using nearest neighbor distances and represents the maximum distance between two points to be considered in the same cluster.
</Note>

## Protected Endpoints

Some endpoints require authentication. Include the JWT token in the Authorization header:

```bash Authenticated Request
curl -X GET "http://localhost:8000/usuarios/" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

**Protected Endpoints:**

- `GET /usuarios/` - List all users
- `GET /usuario/{id}` - Get user by ID
- `PUT /usuario/{id}` - Update user
- `DELETE /usuario/{id}` - Delete user

## Testing with Python

If you prefer Python, here's a complete example using the `requests` library:

```python Python Example
import requests

BASE_URL = "http://localhost:8000"

# 1. Register a user
register_data = {
    "nombre": "María",
    "apellidos": "González López",
    "correo_electronico": "maria.gonzalez@example.com",
    "telefono": "+52 55 9876 5432",
    "contrasena": "SecurePass456!"
}
response = requests.post(f"{BASE_URL}/register", json=register_data)
print("Registration:", response.json())

# 2. Login
login_data = {
    "correo_electronico": "maria.gonzalez@example.com",
    "contrasena": "SecurePass456!"
}
response = requests.post(f"{BASE_URL}/login", json=login_data)
token = response.json()["access_token"]
print("Token:", token)

# 3. Create incident
incident_data = {
    "titulo": "Fuga de agua en tubería principal",
    "descripcion": "Gran fuga de agua desperdiciando recursos",
    "latitud": 19.4284,
    "longitud": -99.1276,
    "estado": "pendiente",
    "usuario_id": 1,
    "categoria_id": 1
}
response = requests.post(f"{BASE_URL}/incidente/", json=incident_data)
print("Incident:", response.json())

# 4. Get ML priorities
response = requests.get(f"{BASE_URL}/incidentes_prioridad")
print("Priorities:", response.json())

# 5. Detect critical zones
response = requests.get(f"{BASE_URL}/ml/zonas-criticas")
print("Critical Zones:", response.json())
```
